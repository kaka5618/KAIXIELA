# 开写啦需求文档

## 1. 项目概述

“开写啦”是一款面向小学 3-6 年级孩子的写作启发工具。孩子输入作文题目后，系统不直接生成范文，而是通过游戏化对话，引导孩子回忆真实经历、观察生活细节、说出自己的感受，并最终整理成可用于动笔的“探险收获卡”。

项目的核心价值是帮助孩子跨过写作文最困难的第一步：不知道写什么、没有素材、无法开头。

## 2. 产品定位

### 2.1 产品一句话

一个陪孩子聊天找作文素材、帮孩子迈出写作第一步的 AI 灵感工具。

### 2.2 目标用户

主要用户：

- 小学 3-6 年级学生
- 有作文题目，但不知道写什么
- 需要通过提问、回忆、提示来找到真实素材

辅助用户：

- 家长
- 可帮助孩子进入小程序、输入题目、查看历史记录
- 但产品语言和交互优先面向孩子

### 2.3 核心场景

孩子拿到作文题目后，例如：

- 我的妈妈
- 难忘的一件事
- 我喜欢的小动物
- 假如我有一支神奇的笔

孩子不是不会写字，而是脑子里没有清晰素材，不知道从哪里开始。此时小程序通过一问一答的方式，帮助孩子把生活中的真实细节挖出来。

## 3. 产品目标

### 3.1 首版目标

- 帮孩子从作文题目中找到可写素材
- 通过对话降低写作启动难度
- 用游戏化方式提高孩子参与意愿
- 输出素材卡片和写作提纲
- 避免直接生成完整范文

### 3.2 非目标

首版不做以下能力：

- 不做完整作文代写
- 不做整篇范文生成
- 不做作文批改
- 不做老师班级管理
- 不做复杂积分、等级、排行榜
- 不做完整写作编辑器

## 4. 核心原则

### 4.1 不代写

AI 只能帮助孩子回忆、整理、启发，不直接替孩子完成作文。

允许输出：

- 素材点
- 关键词
- 写作提纲
- 零散短句
- 表达提示
- 词语替换建议

不允许输出：

- 完整范文
- 可直接提交的完整段落
- 替孩子写好的整篇作文

### 4.2 以孩子为中心

产品语言要适合小学 3-6 年级孩子理解。

语气要求：

- 温和
- 鼓励
- 简短
- 具体
- 像陪伴式引导，而不是批评式教学

### 4.3 真实素材优先

产品重点不是让作文看起来华丽，而是帮助孩子找到真实经历和真实感受。

例如题目是“我的妈妈”，AI 不应直接提供“妈妈像春风一样温暖”这类套话，而应追问：

- 妈妈最常对你说的一句话是什么？
- 妈妈做饭时有什么小动作？
- 有没有一次你觉得妈妈特别辛苦？
- 她生气和高兴时有什么不同？

## 5. MVP 功能范围

## 5.1 首页

首页用于让孩子开始一次灵感探险。

主要元素：

- 产品名称：开写啦
- 主题包装：灵感探险队
- 作文题目输入框
- 年级选择：三年级、四年级、五年级、六年级
- 开始探险按钮
- 历史记录入口

输入字段：

- 作文题目：必填
- 年级：必填
- 作文要求：选填

## 5.2 作文类型识别

孩子输入题目后，系统判断作文类型。

首版支持四类：

1. 写人
2. 记事
3. 状物
4. 想象

识别方式：

- 首版可先使用规则识别
- 后续可接入 AI 分类

规则示例：

- 题目包含“我的妈妈”“我的老师”“一个熟悉的人”，归为写人
- 题目包含“难忘”“一次”“那件事”，归为记事
- 题目包含“小动物”“文具”“玩具”“植物”，归为状物
- 题目包含“假如”“未来”“神奇”“梦”，归为想象

如果无法判断，则进入通用探险流程。

## 5.3 灵感探险对话

对话是产品核心。

系统以“灵感探险队”的方式引导孩子完成多个关卡。每个关卡围绕一个素材目标提问。

### 5.3.1 写人类探险路线

关卡示例：

1. 人物印象
   - 这个人给你的第一感觉是什么？
   - 你最想写他/她哪一点？

2. 语言细节
   - 他/她最常说的一句话是什么？
   - 这句话通常在什么时候说？

3. 动作细节
   - 他/她做某件事时有什么小动作？
   - 这个动作让你想到什么？

4. 具体事件
   - 有没有一件事能说明他/她的特点？
   - 当时发生了什么？

5. 我的感受
   - 这件事之后，你心里有什么感觉？
   - 你想对他/她说什么？

### 5.3.2 记事类探险路线

关卡示例：

1. 事件起点
   - 这件事发生在什么时候？
   - 当时你在哪里？

2. 事情经过
   - 一开始发生了什么？
   - 后来出现了什么变化？

3. 最难忘的画面
   - 你现在最记得哪个画面？
   - 当时谁说了什么、做了什么？

4. 心情变化
   - 你的心情一开始是什么样？
   - 后来发生了什么变化？

5. 收获感悟
   - 这件事让你明白了什么？
   - 如果再发生一次，你会怎么做？

### 5.3.3 状物类探险路线

关卡示例：

1. 外形观察
   - 它长什么样？
   - 颜色、形状、大小有什么特别？

2. 特点功能
   - 它有什么用？
   - 你最喜欢它哪一点？

3. 相关故事
   - 你和它之间发生过什么事？
   - 它陪你度过过什么时刻？

4. 细节描写
   - 摸起来、闻起来、听起来是什么感觉？
   - 它在不同时间有什么变化？

5. 情感连接
   - 为什么它对你重要？
   - 如果它会说话，你觉得它会说什么？

### 5.3.4 想象类探险路线

关卡示例：

1. 想象设定
   - 你的故事发生在哪里？
   - 这个世界和现实有什么不同？

2. 主角设定
   - 主角是谁？
   - 他/她有什么特别能力或愿望？

3. 问题出现
   - 故事里遇到了什么困难？
   - 这个困难为什么重要？

4. 冒险经过
   - 主角做了哪些尝试？
   - 最紧张的地方是什么？

5. 结局和意义
   - 最后问题解决了吗？
   - 这个故事想告诉大家什么？

## 5.4 输入方式

首版支持：

- 文字输入
- 语音输入

文字输入：

- 孩子可以直接打字回答问题

语音输入：

- 孩子可以按住说话
- 系统识别为文字后进入素材整理

语音能力首版可先做为接口预留或基础能力，具体识别服务可后续接入。

## 5.5 游戏化设计

主题名称：

- 灵感探险队

核心游戏化元素：

- 每回答完一个关卡，获得一颗素材宝石
- 每次探险完成，获得一枚轻量徽章
- 不做复杂等级、排行榜、连续打卡

游戏化目标：

- 让孩子愿意继续回答
- 降低写作文的压力
- 让素材收集过程有完成感

## 5.6 探险收获卡

对话结束后，系统生成“探险收获卡”。

收获卡包含：

1. 题目
2. 年级
3. 作文类型
4. 素材宝石
5. 写作提纲
6. 可参考短句
7. 下一步提示

### 5.6.1 素材宝石

素材宝石是孩子回答内容的结构化整理。

示例：

- 人物宝石：妈妈是一个细心的人
- 语言宝石：妈妈常说“慢慢来，别着急”
- 动作宝石：妈妈切菜时总会把袖子卷起来
- 事件宝石：有一次下雨，妈妈送伞到学校
- 感受宝石：我觉得妈妈虽然唠叨，但很关心我

### 5.6.2 写作提纲

提纲只提供结构，不直接写成完整作文。

示例：

- 开头：写出你对妈妈的整体印象
- 中间：写一件能表现妈妈细心的事
- 结尾：写你对妈妈的感谢或理解

### 5.6.3 可参考短句

短句用于启发表达，但不能组成完整范文。

示例：

- “她总是先想到我，再想到自己。”
- “那句话听起来普通，却让我心里暖暖的。”
- “我忽然发现，妈妈的唠叨里藏着关心。”

## 5.7 历史记录

系统默认保存每次探险结果。

历史记录包含：

- 作文题目
- 年级
- 作文类型
- 完成时间
- 素材卡片
- 提纲
- 短句提示
- 获得的徽章

用户能力：

- 查看历史记录
- 删除历史记录
- 重新打开某次收获卡

## 6. 页面结构

首版建议包含 4 个页面：

1. 首页
   - 输入题目
   - 选择年级
   - 开始探险
   - 查看历史

2. 探险对话页
   - 展示当前关卡
   - 展示 AI 问题
   - 支持文字回答
   - 支持语音回答入口
   - 显示已收集素材宝石数量

3. 探险收获卡页
   - 展示题目、类型、年级
   - 展示素材卡片
   - 展示写作提纲
   - 展示短句提示
   - 展示下一步建议

4. 历史记录页
   - 展示历史探险列表
   - 点击查看详情
   - 支持删除

## 8. AI 生成策略

### 8.1 API 接入方式

首版选择通过后端服务接入 AI API。

基本原则：

- 小程序前端不直接调用 AI API
- 小程序前端不保存、不暴露 API Key
- 后端统一封装模型调用、提示词、输出校验和安全边界
- 前端只提交题目、年级、作文要求、当前关卡、孩子回答和历史对话
- 后端返回结构化 JSON，前端按 JSON 渲染问题、素材宝石、提纲和短句

建议链路：

1. 小程序端提交作文题目和年级
2. 后端判断作文类型
3. 后端选择对应探险模板
4. 后端调用 AI API 生成当前关卡问题或追问
5. 孩子回答后，小程序提交回答
6. 后端根据回答继续追问或进入下一关
7. 所有关卡完成后，后端调用 AI API 整理探险收获卡
8. 后端保存历史记录并返回结果

### 8.2 API 输出格式约束

AI API 必须返回结构化 JSON，不允许直接返回自由文本给前端渲染。

对话阶段返回示例：

```json
{
  "mode": "question",
  "type": "写人",
  "stepId": "person_language",
  "stepName": "语言细节",
  "question": "妈妈最常对你说的一句话是什么？",
  "hint": "可以想一想：写作业、吃饭、出门前，她常提醒你什么？",
  "gemName": "语言宝石",
  "allowVoiceInput": true,
  "isFinalStep": false
}
```

收获卡阶段返回示例：

```json
{
  "mode": "result",
  "title": "我的妈妈",
  "grade": 4,
  "type": "写人",
  "gems": [
    {
      "name": "语言宝石",
      "content": "妈妈常说“慢慢来，别着急”，这能表现她温柔、耐心的一面。"
    }
  ],
  "outline": {
    "start": "先写你对妈妈的整体印象。",
    "middle": "重点写一件能表现妈妈耐心的事。",
    "end": "写你从这件事里感受到的爱。"
  },
  "phrases": [
    "这句话很普通，却让我心里安定下来。",
    "她的声音不大，却像在给我加油。"
  ],
  "nextStep": "你可以先从“语言宝石”开始写，把妈妈说这句话时的样子写出来。",
  "badge": "细节发现家"
}
```

### 8.3 系统提示词约束

后端调用 AI API 时，必须注入稳定的系统提示词，确保模型始终遵守产品边界。

系统提示词建议：

```text
你是“开写啦”的小学作文灵感引导助手，不是作文代写工具。

你的服务对象是小学 3-6 年级孩子。你的任务是通过温和、具体、简短的问题，帮助孩子回忆真实经历、观察生活细节、说出自己的感受，并把孩子自己的回答整理成素材卡、写作提纲和少量短句提示。

你必须遵守以下规则：
1. 不生成完整作文。
2. 不生成完整范文。
3. 不生成可直接提交的完整段落。
4. 不替孩子编造没有说过的真实经历。
5. 不使用成人化、模板化、空泛的表达。
6. 不批评孩子，不使用打击式语言。
7. 每次只问一个主要问题。
8. 问题必须适合小学 3-6 年级孩子理解。
9. 如果孩子回答很短，要继续追问更具体的细节。
10. 如果孩子回答“不知道”，要给更小的问题或 2-4 个可选提示。
11. 最终只能输出素材点、提纲、短句提示和下一步建议。
12. 输出必须严格遵守指定 JSON 结构，不要输出 JSON 以外的解释文字。

你可以做：
1. 根据题目判断作文类型。
2. 围绕人物、事件、物品、想象设定进行追问。
3. 把孩子已经说过的内容整理得更清楚。
4. 给出零散短句、词语替换和表达提示。

你不能做：
1. 不能写整段作文。
2. 不能写完整开头、完整结尾或完整主体段。
3. 不能把短句串成一篇文章。
4. 不能为了让作文更好看而编造孩子没有说过的事情。
```

### 8.4 用户意图安全约束

当用户提出代写类请求时，系统必须引导回素材探险。

代写类请求包括：

- 帮我写一篇作文
- 给我一篇范文
- 直接写出来
- 帮我写开头结尾
- 生成一整段
- 写得优美一点，能直接抄

推荐响应策略：

```text
我不能直接替你写完整作文，但我可以帮你把自己的素材找出来。
我们先从一个小问题开始：这篇作文里，你最想写谁/哪件事/哪个东西？
```

如果用户要求完整范文，API 返回：

```json
{
  "mode": "redirect",
  "message": "我不能直接帮你写完整作文，但可以陪你一起找素材。",
  "question": "这个题目里，你第一个想到的人、事或画面是什么？",
  "reason": "avoid_ghostwriting"
}
```

### 8.5 提问策略

AI 提问要遵守：

- 一次只问一个核心问题
- 问题要短
- 问题要具体
- 避免抽象说教
- 孩子回答“不知道”时，要给更小的提示

示例：

不推荐：

- 请描述母亲的精神品质。

推荐：

- 妈妈最近一次让你感动，是在什么时候？

### 8.6 追问策略

当孩子回答很短时：

- 继续追问细节

示例：

孩子回答：

- 妈妈很辛苦。

AI 追问：

- 你是从哪件小事看出妈妈辛苦的？是做饭、上班，还是照顾你？

当孩子回答“不知道”时：

- 给选项提示

示例：

- 没关系，我们换个小问题。妈妈在家里最常做的事是做饭、收拾东西、提醒你学习，还是陪你聊天？

### 8.7 输出策略

AI 输出收获卡时：

- 使用孩子自己的回答作为素材来源
- 不编造孩子没有提到的经历
- 可以润色关键词和短句
- 不生成完整段落
- 不生成完整作文

## 9. 验收标准

### 9.1 功能验收

- 孩子可以输入作文题目和年级
- 系统可以识别作文类型
- 系统可以进入对应探险路线
- 孩子可以逐关回答问题
- 每关完成后可以获得素材宝石
- 完成后可以生成探险收获卡
- 收获卡包含素材、提纲、短句、下一步提示
- 历史记录可以保存、查看、删除

### 9.2 体验验收

- 一个完整流程控制在 5-8 分钟内
- 问题适合小学 3-6 年级理解
- 不出现完整范文
- 不出现大段可直接照抄的作文内容
- 游戏化元素轻量，不干扰写作目标
- 孩子即使回答很短，也能被继续引导

### 9.3 安全边界验收

- 用户要求“直接帮我写一篇作文”时，系统应拒绝生成完整作文，并引导孩子进入素材探险
- 用户要求“给我范文”时，系统应改为提供素材整理和写作思路
- 系统不得虚构孩子没有提供的真实经历
- 历史记录应支持删除

## 10. 后续迭代方向

### 10.1 二稿辅助

孩子写完作文后，可以上传草稿，系统只做局部建议，不整篇代写。

### 10.2 素材库

把孩子多次探险中积累的素材沉淀为个人素材库。

例如：

- 我的家人
- 我的校园
- 我的朋友
- 我的难忘经历

### 10.3 教材题库

接入不同年级常见作文题目，帮助孩子从题库中选择练习。

### 10.4 家长查看

家长可以查看孩子最近完成的灵感探险，但不直接干预对话过程。

### 10.5 写作能力画像

基于历史记录分析孩子常见问题：

- 细节少
- 感受弱
- 事件经过不清楚
- 结尾容易空泛

## 11. 首版优先级

## 12. 当前版本结论

首版产品应聚焦“写前灵感启发”，不要扩展到作文批改、范文生成或完整写作编辑器。

最重要的用户体验是：

1. 孩子输入题目后，不再面对空白页
2. AI 像探险伙伴一样问出具体问题
3. 孩子通过回答找到自己的真实素材
4. 最后拿到一张能帮助动笔的收获卡

这个方向比传统作文批改更适合解决孩子写作中最痛苦的“启动难”问题，也更容易和代写工具区分开。
